🗺️ Roadmap de Desenvolvimento

Plano de ação e evolução do CaraProjetada

✅ Concluído

Fase 1: Baseline

  • Flask app com autenticação AD
  • Conexão VNC reversa
  • Kiosk Chromium
  • Watchdog e Guardian
  • Streaming RTSP
  • Serviços systemd
🔄 Em Andamento

Fase 2: Estabilização

  • Migrar para kernel 6.6+ (via CaraAzul)
  • Substituir xfwm4 por window manager mais leve (openbox?)
  • Adicionar fallback ethernet quando wifi falha
  • Logs centralizados (syslog remoto)
  • Script de backup automático das configurações
📋 Planejado

Fase 3: Multi-Projetor

  • Dashboard central com status de todos os projetores
  • Descoberta automática na rede (mDNS/Bonjour)
  • Configuração remota via API
  • Agendamento de horários (ligar/desligar)
  • Monitoramento de temperatura do SoC
🔒 Planejado

Fase 4: Segurança e Robustez

  • HTTPS com certificado auto-assinado
  • Rate limiting no login
  • Logs de auditoria de acesso
  • Fail2ban para proteção SSH
  • Atualização OTA via git pull
🚀 Futuro

Fase 5: Features Avançadas

  • Suporte a múltiplos usuários simultaneos
  • Streaming de áudio via VNC
  • Modo apresentação (slides + anotações)
  • Compatibilidade com Miracast/AirPlay
  • App mobile para controle

📋 Plano de Ação Atual

│   ├── app.py              # Flask server + VNC control + AD auth

│   ├── totem_watchdog.sh   # Periodic watchdog
│   ├── totem_reset.sh      # Emergency reset
│   ├── start_rtsp.sh       # RTSP camera streaming
│   └── build-kernel.sh     # Kernel building (CaraAzul)
├── systemd/
│   ├── projetor.service    # Flask service (port 80)
│   └── stream-cam.service  # RTSP streaming service
